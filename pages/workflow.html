To generate the susceptibility map we used the classification tool fram the Dzetsaka Plugin. We then converted the classification confidence to class probability using the raster calculator, using this formula
("classification@1" = 2) * "confidence@1" + ("classification@1" = 1) * (100 - "confidence@1")
Sequentially, to validate the output obtained previously, we used the Accuracy Assessment and Sampling, from the processing scripts tools, on the Susceptibility maps and testing point obtained from the previous computation.

  • Producer's accuracy (PA) for a class indicates the likelihood that the class present on the ground is also captured by the classification in the thematic raster.
    • PA = TP / (TP + FN)
  • User's accuracy (UA) for a class reveals how frequently a user of the classified map will encounter the class accurately representing the ground reality.
    • UA = TP / (TP + FP)
  • Overall accuracy (OA) represents the overall percentage of pixels that are correctly classified relative to the total number of pixels.
    • OA = (TP + TN) / (TP + TN + FP + FN)
The value of Overall Accuracy obtained is 0.773, being higher then 0.7, highlights a sufficiently high level of reliability in the results of the susceptibility map.

Exposure Assessment

To conduct the exposure assessment, we acquired and reprojected the population raster dataset to match the coordinate reference system (CRS) of the sub-area vector layer and the susceptibility map obtained in the preceding step.

Employing the Raster calculator, we categorized the susceptibility raster map into four distinct classes. This is the formula we used:
("susceptibility_map@1" < 25) * 1 +  
(("susceptibility_map@1" ≥ 25)AND("susceptibility_map@1" < 50)) * 2 +  
(("susceptibility_map@1" ≥ 50) AND ("susceptibility_map@1" < 75)) * 3 +  
("susceptibility_map@1" ≥ 75) * 4

  • Low: Values in the range [0, 25) %
  • Moderate: Values in the range [25, 50) %
  • High: Values in the range [50, 75) %
  • Very high: Values in the range [75, 100) %
To determine population exposure, a resampling procedure was conducted on the susceptibility map layer, ensuring its resolution conformed with that of the clipped population raster dataset, which approximated 81.67 meters. By utilizing Raster layer zonal statistics, we computed the population count and we subsequencially used the Plugin Plotly to visualize the output through a pie chart illustrating the proportion of population within each susceptibility class before defined.

The exposure assessment constitutes a crucial aspect in our assessment, indeed it aims at identifying and evaluating elements at risk within landslide susceptible zone. The principal aim of an exposure assessment is to quantify and comprehend the vulnerability of elements like the presence of individuals, of buildings, and infrastructure, facilitating the development of effective strategies for risk mitigation.In this exposure assessment in particular we are going to focus on the percentage of people that may be affected by potential landslide By examinating the spatial correlation between areas prone to landslides and exposed elements, we can indentify high-risk zones and assess the potential repercussions of landslides on human activties.

Challenges Faced

  • We had to manually clip the NLZ layer into different sections in order to obtain better distributed points.
  • We had to convert integer into the floating-point data type to successfully merge the NLZ and LS-inventory layers.
  • We used the GDAL tool (Aspect) instead of the SAGA (Slope,aspect,curvature), because otherwise it would indicate a NULL value in flat areas instead of zero, creating gaps within the layer and causing issues for subsequent operations.
  • We launced multiple times the tool to generate random points multiple times in order to produce points consistent with our study area.
